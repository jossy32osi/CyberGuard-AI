# CyberGuard AI - User Stories

## 1. Introduction

This document describes the user stories for CyberGuard AI. These stories capture the needs of different users and explain how the application will help them identify and avoid cyber threats. They serve as a guide for designing and implementing features that solve real-world cybersecurity problems.

---

# 2. Primary Users

The primary users of CyberGuard AI include:

* Students
* Job seekers
* Online shoppers
* Small business owners
* Bank customers
* Social media users
* Elderly internet users
* Parents
* Cybersecurity beginners

---

# 3. User Stories

## Student

**US-001**

**As a student, I want to check whether an email claiming to offer a scholarship is genuine so that I do not become a victim of phishing or identity theft.**

Acceptance Criteria:

* Student can paste an email into the system.
* AI analyzes the email.
* Risk score is displayed.
* Suspicious indicators are highlighted.
* Safety recommendations are provided.

---

**US-002**

**As a student, I want to verify a website before entering my university login credentials so that my account remains secure.**

Acceptance Criteria:

* User pastes a URL.
* System checks for phishing indicators.
* AI explains the findings.
* User receives recommendations.

---

## Job Seeker

**US-003**

**As a job seeker, I want to analyze recruitment emails and job offers so that I can avoid employment scams.**

Acceptance Criteria:

* User submits job email.
* AI detects suspicious language.
* Fake interview requests are flagged.
* User receives a risk score.

---

**US-004**

**As a job seeker, I want to verify company websites before submitting my CV so that my personal information is protected.**

Acceptance Criteria:

* URL scanning is available.
* Suspicious domains are identified.
* Safe browsing advice is displayed.

---

## Online Shopper

**US-005**

**As an online shopper, I want to verify shopping websites before making payments so that I avoid fraudulent online stores.**

Acceptance Criteria:

* Website analysis is performed.
* Scam indicators are identified.
* Overall trust rating is displayed.

---

## Bank Customer

**US-006**

**As a bank customer, I want to analyze SMS messages requesting account verification so that I do not disclose sensitive banking information to criminals.**

Acceptance Criteria:

* SMS text can be pasted.
* AI detects phishing language.
* Risk score is displayed.
* Warning messages are generated.

---

## Small Business Owner

**US-007**

**As a small business owner, I want to verify emails received from suppliers and customers so that I avoid business email compromise attacks.**

Acceptance Criteria:

* Email content is analyzed.
* Suspicious requests are detected.
* AI explains possible risks.

---

## Parent

**US-008**

**As a parent, I want to check suspicious messages received by my children so that I can help protect them from online scams.**

Acceptance Criteria:

* Parent submits text.
* AI analyzes threats.
* Recommendations are easy to understand.

---

## Elderly Internet User

**US-009**

**As an elderly internet user, I want CyberGuard AI to explain risks in simple language so that I understand whether something is dangerous without needing technical knowledge.**

Acceptance Criteria:

* Plain language explanations.
* Large and easy-to-read interface.
* Clear recommendations.

---

## Cybersecurity Beginner

**US-010**

**As a beginner in cybersecurity, I want CyberGuard AI to explain why a website or message is considered suspicious so that I can learn how cybercriminals operate.**

Acceptance Criteria:

* AI explains detected indicators.
* Educational tips are displayed.
* Threat categories are shown.

---

# 4. Administrative User Stories

## Administrator

**US-011**

**As an administrator, I want to monitor application performance so that I can maintain reliability and improve the system over time.**

Acceptance Criteria:

* View application logs.
* Monitor scan statistics.
* Monitor system health.

---

**US-012**

**As an administrator, I want to update threat detection rules so that CyberGuard AI can recognize new scam techniques.**

Acceptance Criteria:

* Update threat database.
* Manage AI configuration.
* Deploy improvements securely.

# 7. Definition of Done

A user story is considered complete when:

* The feature has been implemented.
* Testing has passed successfully.
* Security requirements have been verified.
* Documentation has been updated.
* The feature is approved for release.

# 8. Summary
